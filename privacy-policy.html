Сведения об операторе и контактах

Настоящая Политика конфиденциальности описывает, какие данные обрабатываются при использовании мобильного приложения «Просто Бюджет: Домашний учёт» для Android (далее — «Приложение»), для каких целей они используются, кому могут передаваться и как пользователь может управлять обработкой данных.

1. Общие положения

Приложение предназначено для личного и семейного учёта доходов, расходов, переводов между кошельками, бюджетов, целей, долгов и повторяющихся операций.

Приложение работает без обязательной регистрации, без аккаунта, без банковских интеграций и без подключения к банковским счетам. Приложение не читает SMS, уведомления банков, контакты, геолокацию, камеру, микрофон, фотографии или историю звонков.

Финансовые записи пользователя хранятся локально на устройстве пользователя. Разработчик не получает доступ к этим финансовым данным и не хранит их на собственном сервере.

Правовые основания обработки данных зависят от конкретной функции: действия пользователя при вводе и хранении данных в Приложении, исполнение пользовательского соглашения, согласие или настройка пользователя для рекламы и диагностики, а также требования применимого законодательства.

2. Какие данные хранятся локально на устройстве

Приложение может хранить на устройстве следующие данные, которые пользователь вводит самостоятельно:

  • доходы, расходы и переводы;
  • суммы операций;
  • кошельки и счета;
  • категории;
  • комментарии к операциям;
  • бюджеты и лимиты;
  • цели накоплений;
  • долги;
  • повторяющиеся операции;
  • настройки приложения;
  • данные PIN-защиты в виде технического хэша, без хранения исходного PIN-кода.

Эти данные используются только для работы функций Приложения: отображения баланса, истории операций, статистики, бюджетов, целей, долгов, экспорта и резервного копирования.

Android cloud/device backup для базы, настроек и файловой области Приложения отключён. Перенос данных на другое устройство выполняется вручную через экспорт и восстановление резервной копии, если пользователь сам выберет такую функцию.

3. Экспорт и резервные копии

Приложение может позволять пользователю самостоятельно создавать CSV-экспорт или JSON-резервную копию.

Такие файлы создаются только по действию пользователя. После сохранения, отправки или передачи файла пользователь самостоятельно отвечает за его хранение и передачу третьим лицам.

Разработчик не получает копии экспортированных файлов и не имеет доступа к ним.

4. Реклама и Yandex Mobile Ads

Приложение содержит рекламу, которая показывается с использованием Yandex Mobile Ads SDK.

Реклама может показываться на основных некритичных экранах Приложения. Реклама не должна показываться внутри форм добавления или редактирования операций, в процессе экспорта, восстановления, удаления данных, на экране PIN-защиты и в onboarding.

Для показа рекламы Yandex Mobile Ads SDK и связанные рекламные сервисы могут обрабатывать технические и рекламные данные, например:

  • IP-адрес;
  • рекламный идентификатор Android, если он доступен на устройстве;
  • сведения об устройстве и операционной системе;
  • сведения о приложении;
  • тип интернет-соединения;
  • данные о загрузке, показе и взаимодействии с рекламой;
  • технические данные, необходимые для показа, ограничения, измерения, диагностики и улучшения рекламы.

Финансовые данные пользователя, включая суммы операций, балансы, комментарии, названия кошельков и пользовательские категории, не передаются в Yandex Mobile Ads SDK разработчиком Приложения.

Пользователь может ограничить персонализацию рекламы в настройках Приложения, если такая настройка доступна, а также в системных настройках Android, связанных с рекламным идентификатором. Отключение персонализации не обязательно отключает все рекламные блоки.

5. Техническая диагностика и Yandex AppMetrica

Приложение может использовать Yandex AppMetrica для технической диагностики, статистики запусков и улучшения стабильности, если такая функция включена в опубликованной сборке и пользователь не отключил диагностику в настройках Приложения.

Yandex AppMetrica может обрабатывать:

  • технические сведения об устройстве, операционной системе и версии Приложения;
  • сведения о запуске Приложения;
  • технические идентификаторы приложения или устройства, если они доступны SDK;
  • диагностические данные, необходимые для работы SDK и оценки качества Приложения.

В текущей логике Приложения собственные аналитические события ограничиваются техническими событиями без пользовательских финансовых данных, например фактом запуска Приложения.

В Yandex AppMetrica не передаются суммы операций, балансы, комментарии, названия кошельков, пользовательские категории, PIN-коды, резервные копии или точная финансовая история пользователя.

Пользователь может отключить диагностику в настройках Приложения. Локальная статистика и основные функции учёта продолжат работать.

6. Разрешения Android

Приложение может использовать следующие разрешения Android:

  • доступ к интернету — для загрузки и показа рекламы, а также для работы технической диагностики SDK, если она включена;
  • доступ к состоянию сети — для корректной работы рекламного SDK и технической диагностики;
  • рекламный идентификатор Android — для показа, ограничения и измерения рекламы, если он доступен на устройстве.

Приложение не запрашивает доступ к SMS, уведомлениям банков, контактам, геолокации, камере, микрофону, фотографиям, звонкам или банковским данным.

7. Передача данных третьим лицам

Разработчик не продаёт пользовательские данные.

Финансовые данные, введённые пользователем в Приложении, не передаются разработчиком третьим лицам.

Технические и рекламные данные могут обрабатываться сторонними SDK и сервисами:

  • Yandex Mobile Ads SDK — показ рекламы, персонализация рекламы при наличии соответствующей настройки, подсчёт показов и диагностика рекламных блоков;
  • Yandex AppMetrica — техническая диагностика и статистика работы Приложения, если диагностика включена;
  • RuStore и Google Play — техническая обработка данных, связанная с установкой, обновлением и работой магазина приложений, в соответствии с правилами соответствующего магазина;
  • почтовые сервисы — обработка данных письма, если пользователь сам обращается в поддержку по электронной почте.

Обработка данных сервисами Яндекса регулируется документами Яндекса:

  • Политика конфиденциальности Яндекса: https://yandex.ru/legal/confidential/
  • Yandex Mobile Ads SDK Agreement: https://yandex.ru/legal/mobileads_sdk_agreement/
  • Yandex AppMetrica: https://appmetrica.yandex.ru/docs/ru/sdk/android/analytics/android-operations

Если пользователь обращается в поддержку по электронной почте, разработчик может получить адрес электронной почты, имя, текст обращения и другие сведения, которые пользователь сам укажет в письме. Эти данные используются только для ответа на обращение и решения вопроса пользователя. Не отправляйте в поддержку финансовые записи, суммы, балансы, PIN-коды и резервные копии без необходимости.

8. Хранение и удаление данных

Финансовые данные хранятся локально на устройстве пользователя до тех пор, пока пользователь не удалит их в Приложении, не очистит данные Приложения средствами Android или не удалит Приложение с устройства.

Так как Приложение не использует аккаунт и не хранит финансовые данные на сервере разработчика, разработчик не может удалённо просматривать, изменять или удалять локальные финансовые данные с устройства пользователя.

Пользователь может самостоятельно удалить локальные данные через функции Приложения, настройки Android или удаление Приложения.

Данные, которые обрабатываются SDK Яндекса, хранятся и удаляются по правилам соответствующих сервисов Яндекса.

Обращения в поддержку хранятся в течение срока, необходимого для ответа на обращение и защиты законных интересов разработчика, если более длительный срок не требуется по закону.

9. Безопасность

Приложение использует локальное хранилище Android. Данные Приложения изолированы средствами операционной системы Android.

PIN-код, если пользователь включает PIN-защиту, не хранится в исходном виде. Для проверки PIN используется техническое представление в виде хэша.

Приложение отключает автоматическое облачное резервное копирование данных Приложения средствами Android. Пользователь может переносить данные вручную через резервную копию, если сам выберет экспорт.

Приложение не предназначено для замены системной защиты устройства, шифрования устройства или банковских средств защиты.

10. Дети

Приложение не предназначено специально для детей и не является детским приложением.

Приложение не содержит функций социальной сети, публичной публикации пользовательского контента или общения между пользователями.

  • отключить диагностику Приложения;
  • сбросить рекламный идентификатор или ограничить его использование в системных настройках Android;
  • удалить локальные данные через функции Приложения или настройки Android;
  • удалить Приложение и его локальные данные;

Поскольку финансовые данные хранятся локально на устройстве пользователя и не передаются разработчику, разработчик не имеет технической возможности просматривать, изменять или удалять такие данные удалённо.
